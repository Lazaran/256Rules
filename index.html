Rule 0

|| Binary 00000000

Combos:

000 : 001 : 010 : 011 :
100 : 101 : 110 : 111 :

This program is an implementation of Cellular Automata.

It requires two things to run:

  • A 1-dimensional array of 1s and 0s (Array)
  • A rule with all possible outcomes of an n-long combination, where n is some number(Rule)

The program will iterate over every value in the Array while also looking at the values to the left and right of the current.

These 3 digits, each of which can either be a 1 or a 0, give 8 possible combinations, each one of which is covered in the Rule.

A Rule is an 8 digit long array, each digit represents the outcome for the combo defined at its index position (so the first value in the Rule array is for the first combo '000')

The following list is the order in which I've listed the combos defined in a Rule.

  1. 000
  2. 001
  3. 010
  4. 011
  5. 100
  6. 101
  7. 110
  8. 111

The Rule defines the outcome of any combo as a 1 or a 0, which is then passed into a new 1-dimensional array to start the process over again.

For example, say we have a Rule '01011010' and a combo of '010' shows up in the Array. We take the combo and compare it against each in the above list, until one matches.

While going down the list, starting at zero, we count how many combos we pass until we match. The number we have is the index of that combo.

We then compare that to the Rule, starting at 0, we count along until the index of the combo matches our position on the Rule.

The value at this position is the outcome of that combo with that Rule. This process is done over 500 times per line in this simulation, all very very quickly.

There are a total of 256 Rules and they are commonly referred to by their Wolfram Code (invented by Stephen Wolfram (certified mathematical badass)) which simply uses the binary value of the Rule. For example, the code '01011010' is Rule 90 because its binary value is 90.
